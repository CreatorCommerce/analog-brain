title: "Accessing Vs Creating Supply"
description: "How to decide whether Analog should access existing IRL supply or create new host-side supply."

---

Analog should not treat all supply work as the same job. Sometimes the right motion is to access existing IRL supply. Sometimes the right motion is to create new supply because the market does not have enough usable hosts, spaces, operators, or sponsor surfaces yet.

The seed-deck rule is simple: IRL demand is already exploding, and it has been mostly bootstrapped. Funding is the unlock that reduces burnout, reactivates dormant communities, raises event quality, and converts more first-timers into regulars.

## Access Existing Supply

Use this motion when the event, community, space, or operator already exists and can become sponsor-ready with packaging.

Signals:

- recurring attendance or strong one-time demand
- clear host or owner
- known audience or category
- sponsor permission already exists
- RSVP, ticketing, check-in, or claim data can be captured
- the host can execute with a brief and spend card
- the host already has enough trust that funding improves the ritual instead of manufacturing it

Actions:

1. Claim or create the catalog record and host profile.
2. Capture sponsor boundaries and category exclusions.
3. Add proof surfaces: attendance, social, content, claims, or prior sponsors.
4. Package the sponsor surface.
5. Route into a sponsor campaign, RFE, or host-side Request for Sponsorship.

## Create New Supply

Use this motion when sponsor demand exists but the right host-side option does not.

Signals:

- sponsors want a category, city, or audience but no qualified host appears
- local communities exist but do not have sponsor surfaces yet
- spaces exist but no operator owns programming
- hosts want to do something but lack venue, vendor, insurance, or funding
- a city lead can assemble the missing parties
- a strong community exists but is stuck as a side hustle because organizers are fronting money, chasing reimbursements, or absorbing unpaid coordination work

Actions:

1. Identify the missing asset: host, space, vendor, insurance, operator, sponsor, or proof.
2. Recruit or create the missing asset through grants, city leads, owned activations, partner lists, or dot connectors.
3. Start with a smaller activation that can prove the operating model.
4. Convert the activation into a reusable host package.
5. Create a no-fronting-money path when possible: approved funds, scope, spend card, proof expectations, and operator upside.
6. Score it with the testing grid before asking sponsors for larger budget.

## Running Vertical Playbook

Use running and adjacent movement communities as the first verticalized supply play when the city has enough club density.

The sequence:

1. Invite local clubs to apply to a sponsor or category fund.
2. Approve best-fit clubs based on trust, cadence, spend use, and proof readiness.
3. Fund with branded spend cards or store credit.
4. Activate obvious spend categories: coffee, race entries, recovery, photography, perks, samples, and meetup supplies.
5. Prove spend, attendance, category lift, social moments, and community sentiment.
6. Renew top operators and let them earn more.

This creates a reusable proof loop before Analog generalizes into every community category.

## Use Prospecting When Supply Exists Off-Platform

Use event set prospecting when the sponsor has a clear event set and the right hosts probably exist, but they are not on Analog yet.

Actions:

1. Create the campaign and event set.
2. Define the market persona and special requests.
3. Build a prospect list from the [Event Prospecting Catalog](/market/event-prospecting-catalog), public discovery, partner lists, city lead knowledge, or sponsor-known communities.
4. Route outreach back to the campaign application link.
5. Convert responses into applications, collabs, catalog updates, or asset-need notes.

## Decision Rule

| Question | If yes | If no |
|---|---|---|
| Does a trusted host already exist? | Access supply | Create or recruit host supply |
| Is there a natural sponsor surface? | Package it | Design one with the host |
| Can we capture proof? | Run a pilot | Add hosted page, claim flow, check-in, or manual proof first |
| Can this repeat? | Score for unlock | Treat as a learning activation |
| Is the blocker local? | Use a city lead | Solve through product, sponsor, or operator motion |
| Is the blocker unknown? | Log an asset-need note | Keep researching until the missing asset is clear |
| Would funding create more good supply? | Create a fund, grant, or operator path | Access and package existing supply first |
