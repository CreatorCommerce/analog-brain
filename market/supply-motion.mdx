---
title: "Accessing Vs Creating Supply"
description: "How to decide whether Analog should access existing IRL supply or create new host-side supply."
pageType: "How-to"
---

Analog should not treat all supply work as the same job. Sometimes the right motion is to access existing IRL supply. Sometimes the right motion is to create new supply because the market does not have enough usable hosts, spaces, operators, or sponsor surfaces yet.

## Access Existing Supply

Use this motion when the event, community, space, or operator already exists and can become sponsor-ready with packaging.

Signals:

- recurring attendance or strong one-time demand
- clear host or owner
- known audience or category
- sponsor permission already exists
- RSVP, ticketing, check-in, or claim data can be captured
- the host can execute with a brief and spend card

Actions:

1. Claim or create the host profile.
2. Capture sponsor boundaries and category exclusions.
3. Add proof surfaces: attendance, social, content, claims, or prior sponsors.
4. Package the sponsor surface.
5. Route into a sponsor campaign, RFE, or host-side Request for Sponsorship.

## Create New Supply

Use this motion when sponsor demand exists but the right host-side option does not.

Signals:

- sponsors want a category, city, or audience but no qualified host appears
- local communities exist but do not have sponsor surfaces yet
- spaces exist but no operator owns programming
- hosts want to do something but lack venue, vendor, insurance, or funding
- a city lead can assemble the missing parties

Actions:

1. Identify the missing asset: host, space, vendor, insurance, operator, sponsor, or proof.
2. Recruit or create the missing asset through grants, city leads, owned activations, partner lists, or dot connectors.
3. Start with a smaller activation that can prove the operating model.
4. Convert the activation into a reusable host package.
5. Score it with the testing grid before asking sponsors for larger budget.

## Decision Rule

| Question | If yes | If no |
|---|---|---|
| Does a trusted host already exist? | Access supply | Create or recruit host supply |
| Is there a natural sponsor surface? | Package it | Design one with the host |
| Can we capture proof? | Run a pilot | Add hosted page, claim flow, check-in, or manual proof first |
| Can this repeat? | Score for unlock | Treat as a learning activation |
| Is the blocker local? | Use a city lead | Solve through product, sponsor, or operator motion |
