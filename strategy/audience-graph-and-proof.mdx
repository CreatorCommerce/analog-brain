Analog should become an IRL audience allocation layer, not only a recap tool. The strategic question is:

```text
Which events, communities, venues, and creator-led moments look like the sponsor's best customers or can grow the category the sponsor cares about?
```

## Core Doctrine

Hosted invites, RSVPs, check-ins, ticketing fields, benefit claims, and attendance imports are not side utilities. They are the data surfaces that make Analog more valuable each time real-world activity happens.

The proof loop is:

```text
RSVP / ticket / check-in / claim -> normalized identifier -> sponsor-side outcome -> attribution window -> confidence tier -> renewal recommendation
```

Direct attribution is a core proof lane, but market-growth underwriting comes first. The strongest early question is whether the activation increased participation, category conversation, content velocity, repeat ritual strength, and sponsor-ready supply.

## IRL Culture Graph

Analog should build toward the data layer that makes IRL culture useful to sponsors without flattening communities into ad inventory.

The analogy:

| Existing data layer | What it explains | Analog equivalent |
|---|---|---|
| ZoomInfo-style B2B data | Which companies, contacts, and accounts exist. | Which communities, venues, hosts, spaces, operators, and city scenes exist. |
| Meta-style social and creator data | Which creators, audiences, and digital attention paths exist. | Which real-world rituals, rooms, cultural operators, and trusted gatherings can create audience potential. |

Analog's wedge is the motion layer: events are fluid, not static. The useful data is cadence, city, scene, host reliability, sponsor permission, audience hypothesis, cultural momentum, content velocity, spend behavior, proof readiness, and renewal outcome.

<Check>
The goal is not raw attendee resale. The goal is privacy-safe IRL market intelligence that helps sponsors decide what to fund, helps hosts prove what they can move, and helps Analog learn where the next dollar should go.
</Check>

## Capture Paths

| Capture path | Best use | Practical rule |
|---|---|---|
| Hosted event page | Clean RSVP, ticket, consent, check-in, benefit claim, and post-registration flow | Prefer this when Analog controls the invite or can co-host the page. |
| Attendance import | External event platforms, member lists, Luma/Eventbrite/Partiful exports, CSVs, venue guest lists | Import only with scoped permission and consent source. |
| Physical location claim | Discounts, samples, class credits, lane credits, venue benefits, retail or hospitality offers | Use QR/print/link flows to capture claim and redemption state. |
| Social proof | Connected account posts, story mentions, UGC, creator/host posts, recap assets | Treat as market-growth and sponsor-safe proof, not raw conversion proof by itself. |
| Sponsor-side outcome import | Shopify, CRM, POS, booking, loyalty, lead, app install, reservation, or CSV/webhook events | Match privately; do not distribute raw attendee lists. |

## Audience Intelligence Products

| Product | Who uses it | What it answers |
|---|---|---|
| Event audience profile | Host and sponsor | Who attends, what categories they likely care about, and what proof exists. |
| Brand audience match | Sponsor | Which events or communities resemble the sponsor's best customers. |
| Lookalike event recommendations | Sponsor, operator, city lead | Which upcoming or recurring opportunities resemble prior converting audiences or target customers. |
| Existing customer overlap | Sponsor | Whether the event already contains privacy-safe customer overlap that can create authentic referral energy. |
| Scene benchmark | Sponsor, operator, public fund | Whether this category/city/event type over-indexes on participation, check-in, claims, or sponsor proof. |

## Privacy Rules

Protect trust before optimizing proof:

- Do not sell or expose raw attendee lists as the product.
- Report overlap counts, percentages, tiers, and explanations before exposing row-level detail.
- Blur or threshold tiny overlap cohorts unless there is explicit permission to act.
- Let existing customers opt into ambassador, referral, bring-a-friend, or featured-customer flows.
- Keep consent source, data provenance, and attribution window attached to the proof.
- Use raw PII only inside scoped matching workflows where legally and commercially allowed.

## Testing Questions

Use this page when a pilot involves attendee capture, hosted pages, imports, or low-touch attribution.

Score the pilot on:

- Will the host use Analog capture or provide a clean import?
- Does the sponsor have an outcome source that can match later?
- Does the audience proof change sponsor willingness to fund or renew?
- Does overlap or lookalike fit improve the sponsor brief?
- Does the proof protect host and attendee trust?
- Can the same capture pattern repeat without bespoke operator work?
