Analog should run two motions at the same time.

The seed deck's market-formation thesis is that IRL community was not manufactured by capital. It arrived through demand. Analog expands the number of IRL communities that can exist by making funding, proof, and renewal work.

## Motion 1: Selective Sponsor-Funded Pilots

These are controlled tests with serious sponsors.

They prove:

- sponsor demand
- spend governance
- host execution quality
- proof quality
- renewal logic
- which use cases deserve more product build

The product path is:

```text
Sponsor budget -> campaign -> event set or creator/community cohort -> brief -> host event or location activation -> spend watch -> proof -> unlock recommendation
```

## Motion 2: Market Formation

This motion creates sponsor-ready supply before the marketplace is liquid.

It can use:

- grants
- Requests for Events
- Requests for Communities
- hosted sponsor forms
- Play Fair
- city leads
- owned Analog activations
- partner APIs
- host-side Requests for Sponsorship
- Event Prospecting Catalog records
- Asset Need Intelligence notes

Market formation proves:

- who exists
- who can execute
- which cities have missing spaces, hosts, insurance, vendors, or operators
- what sponsors want but cannot source yet
- what new supply should be created

## Funding Curve Rule

Funding is not just a reward after a community succeeds. Funding can create the next stage of supply by reducing burnout, reactivating dormant clubs, raising event quality, and helping first-timers become regulars.

Use this rule when evaluating a market:

| Market state | Analog response |
|---|---|
| Existing trusted community, weak economics | Add sponsor funding, spend controls, proof, and operator upside. |
| Sponsor demand, no obvious host | Use city leads, grants, RFEs, RFCs, or owned activations to create supply. |
| Community demand, poor sponsor fit | Protect the community and search for a better sponsor category. |
| Strong behavior, no trust layer | Make the new funding behavior feel safe and normal through approval, transparency, and proof. |

## How They Compound

```text
Pilot proof -> clearer sponsor thesis -> better RFEs and grants -> richer host supply -> better pilots -> stronger reports -> more sponsor budget
```

## Product Rule

The two motions should converge into the same product objects. A grant applicant should be able to become catalog supply. A city lead discovery should be able to become a host record. A host-side sponsorship package should be able to become a sponsor campaign. A completed pilot should become a reusable playbook.

## Producer Bench Rule

Early pilots and owned activations need real operators. The producer bench should not become an open-ended services trap; its job is to:

- run early activations well enough to earn trust
- reveal where product objects are missing
- convert manual runbooks into repeatable docs, tasks, files, proofs, and templates
- protect community taste while satisfying sponsor requirements
- teach Analog which city/category motions are actually repeatable
