This use case came from founder-supplied call context with an operator who had run a presidential Nevada campaign and later worked in the White House. The energy was understandably perspective-biased, but strategically useful: political and advocacy work runs into the same local leaders, spaces, volunteer captains, cultural organizers, and community trust constraints as SaaS, eComm, and civic/nonprofit work.

Read this after the other use-case pages. Political action is the highest-control lane, so it should come after the reader understands the core Sponsor, Host, spend, proof, and trust model.

The Analog lesson is not "become a political tool first." It is:

```text
Own trusted local supply and regional intelligence
-> route different Sponsor lanes into that supply
-> govern spend and proof by lane
-> learn which Hosts, cities, and communities deserve the next dollar
```

Political action is a useful stress test because the operating map is explicit: national, state, district, county, city, ward, Host, spend, proof, and next allocation.

## Market Use Case

A political campaign, PAC, advocacy organization, ballot initiative, party committee, or donor coalition funds trusted local Hosts to run grassroots gatherings as part of a broader campaign.

```text
Political Action Sponsor
-> national/state/district/county/city/ward event sets
-> local Hosts and volunteer leaders
-> governed field spend
-> attendance, action, sentiment, and proof
-> next allocation or reallocation
```

The key use case is the political funding cascade:

```text
national
-> state
-> district
-> county
-> city
-> ward/precinct/neighborhood
-> Host event
```

Analog's role is not to make money move loosely through that chain. The role is to turn each layer into a controlled budget envelope with the right Sponsor authorization, allowed uses, local recipients, proof requirements, and reallocation logic.

| Layer | What happens |
|---|---|
| Sponsor | Campaign, PAC, advocacy organization, party committee, ballot initiative, or donor coalition funds a grassroots activation program. |
| Event sets | National, state, district, county, city, ward, precinct, issue, language/community, audience, or field-office lane. |
| Hosts | County captains, volunteer leaders, community Hosts, campus organizers, local operators, advocacy partners, venue owners, and trusted dot connectors. |
| Spend | Room minimums, food, coffee, printing, tabling supplies, accessibility, translation, local media, basic production, event supplies, and approved field costs. |
| Community value | Supporters and reachable communities get useful rooms, local belonging, practical information, and easier participation. |
| Sponsor value | The campaign sees where grassroots dollars create real participation, volunteer energy, local trust, and repeatable field action. |

All spend categories, data use, and Sponsor/Host relationships need counsel-reviewed campaign-finance, election-law, privacy, and platform-policy controls before production use.

## Funding Cascade

Political campaigns already think in nested geographies. Analog should turn that structure into product objects.

| Political layer | Analog object | What it decides | What proof rolls up |
|---|---|---|---|
| National campaign, national committee, donor coalition, or major advocacy fund | Sponsor and top-level campaign | Which states, issues, populations, or races deserve field activation budget. | State and district performance, risk, spend pace, and allocation recommendation. |
| State campaign, state party, statewide ballot program, or state advocacy team | Campaign segment or event set family | Which districts, counties, cities, coalitions, and field offices need local activation. | County/city traction, local Host quality, volunteer or action proof where permitted. |
| District | Event set | Which congressional, state legislative, school board, judicial, or issue district needs targeted rooms. | District-level participation, field learning, and whether a district should receive more budget. |
| County | Event set or collab cohort | Which county captains, operators, venue partners, or coalition groups can deploy budget responsibly. | County spend stewardship, attendance ranges, organizer reliability, local blockers. |
| City | Event set, Host cohort, or city fund | Which city communities, venues, campuses, faith/civic groups, neighborhood leaders, and issue rooms matter. | City-level participation, sentiment, media or mention signal where appropriate, supply gaps. |
| Ward, precinct, or neighborhood | Collab and Host event | Which trusted local Host should run the actual gathering or field-support moment. | Event completion, receipts, attendance/action proof where permitted, Host note, risk notes. |

This is where Analog can be materially different from political consulting, ad buying, and spreadsheets. The product can show which local layer actually absorbed the dollar and which layer only looked good on a plan.

## How Money Should Move

Treat the cascade as governed allocation, not casual pass-through.

| Step | Product rule |
|---|---|
| 1. Define legal Sponsor boundary | The campaign, committee, PAC, ballot entity, advocacy organization, party committee, or donor coalition is recorded with funding source, authorized users, compliance owner, and reporting boundary. |
| 2. Create geography envelopes | Budget is split into state, district, county, city, ward, precinct, issue, or audience envelopes. |
| 3. Define approved uses | Each envelope has allowed spend, restricted spend, timing, geography, recipient type, receipt rules, and proof expectations. |
| 4. Approve local recipients | Hosts, vendors, venues, captains, or operators are approved into collabs before money is accessible. |
| 5. Release scoped budget | Funds are deployed through event-scoped or lane-scoped controls, such as spend cards, invoices, reimbursements, or future treasury rails where appropriate. |
| 6. Capture proof | Receipts, event completion, attendance range, action proof where permitted, Host notes, risk notes, and field learnings attach to the local event. |
| 7. Roll up learning | Proof rolls up from event to ward, city, county, district, state, and national view. |
| 8. Reallocate | Strong Hosts, cities, wards, and event types get more budget; weak or risky lanes pause. |

The product question after every tranche:

```text
Which geography, Host, event type, and local operator deserves the next field dollar?
```

## Marketplace Logic

Political action should use the same marketplace doctrine as other Sponsor lanes, but with stricter controls.

| Marketplace question | Political version |
|---|---|
| Right dollar | Is this micro budget, macro Host support, venue spend, print, hospitality, translation, accessibility, or field-supply money? |
| Right Host | Which local operator has trust, execution ability, and permission to gather people around this objective? |
| Right event type | House party, volunteer training, ballot explainer, town hall, campus gathering, coalition dinner, canvass launch, phone bank, or issue night. |
| Right geography | State, district, county, city, ward, precinct, campus, neighborhood, or coalition. |
| Right visibility | Private link-only for sensitive or controlled programs; ICP-targeted for specific Hosts, communities, or geographies; broadly marketed only when counsel, platform rules, and campaign strategy support it. |
| Right benchmark | Compare against political action proof: compliant spend, local participation, permitted action proof, privacy-safe sentiment, and allocation learning. |

AI can help rank event types, local Hosts, and geography envelopes by receptiveness, comparable proof, and risk. Human operators and campaign compliance owners still approve sensitive fit, messaging, spend, and data use.

## What It Proves

| Proof question | Why it matters |
|---|---|
| Can local Hosts run compliant grassroots events with small controlled budgets? | Campaigns usually struggle to know which distributed organizers can use money responsibly before the work happens. |
| Can spend be optimized from national to state to district to county to city to ward to event? | Analog can show where grassroots budget is creating participation instead of only tracking top-down media or staff spend. |
| Can upstream funders see the local use of dollars without overexposing people? | Political Sponsors need accountability, but raw voter, donor, volunteer, attendee, and affiliation data should remain protected. |
| Which communities and leaders overlap with other lanes? | The same trusted organizers may matter to civic, SaaS/tech, eComm, and nonprofit Sponsors under different proof rules. |
| Can proof stay useful without becoming invasive? | Political action requires especially careful privacy, consent, and sensitive-affiliation boundaries. |
| Does a city department become easier to fund? | If a campaign needs deeper local infrastructure, it may justify a stronger Analog city layer, local operator bench, or grant pool. |

## Proof Standards

Political proof should be compliance-safe and purpose-limited. Early proof can include:

- budget used versus approved amount
- receipts and merchant categories
- event happened and Host completed required tasks
- attendance range or check-in count where permitted
- volunteer shifts, signups, calls, doors, or issue actions only when lawfully captured and permissioned
- Host note on local sentiment and blockers
- ward/city/county/district/state rollup for what should repeat
- risk notes for privacy, safety, community fit, and Sponsor presence

Do not sell or expose raw political-affiliation data as market intelligence. Do not make sensitive voter, donor, volunteer, or attendee data visible to Sponsors beyond the authorized campaign/program context.

## Data Boundary

Political data is sensitive even when the activation is public.

| Data type | Default handling |
|---|---|
| Voter file, donor list, volunteer list, or political affiliation | Do not expose as general marketplace intelligence. Use only inside authorized campaign/program context. |
| Attendance and check-in data | Capture only with clear permission, purpose, and retention rules. Share rollups by default. |
| Action proof | Capture only when legally allowed and campaign-approved. Prefer counts, tiers, and completion status before raw person-level detail. |
| Host and Venue mentions | Use only when authorized and campaign-relevant. Do not pressure Hosts or Venues into political speech. |
| Field notes | Separate useful allocation learning from sensitive community context. Redact or summarize before Sponsor-wide sharing. |

## Use When / Avoid

Use this motion when the Sponsor has a lawful field budget, a clear geography, trusted local Hosts, approved spending rules, and a proof standard that can inform the next allocation without exposing sensitive people or communities.

Avoid it when the goal is opaque influence, off-platform money movement, evasion of campaign-finance rules, raw voter-data extraction, aggressive persuasion metrics inside sensitive communities, or political activity that would damage Analog's trust in a city market.

## Operating Standard

Analog should only pursue this lane if the program can be structured as:

```text
authorized Sponsor
-> scoped geography envelope
-> approved Host or vendor
-> controlled spend
-> permissioned proof
-> privacy-safe rollup
-> next-dollar decision
```

If any layer cannot be made legible, do not fund it through Analog.
