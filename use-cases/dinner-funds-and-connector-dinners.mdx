Dinner is one of the cleanest Analog formats. A Sponsor funds a trusted Host, the Host creates a room people actually want to be in, the community receives real hospitality, and proof can be captured through spend, attendance, photos where natural, Host notes, and follow-up.

This page covers two founder-supplied use cases:

1. John Deere funds a trusted Iowa restaurant owner to bring farmers together.
2. Tapcart funds eCommerce legends and dot connectors to run dinners across markets.

```text
Sponsor funds trusted Host
-> Host curates the room
-> spend is governed
-> community receives hospitality
-> Sponsor gets proof
-> best Hosts unlock more budget
```

## Use Case 1: John Deere Farmer Dinner Fund

John Deere sponsors an Iowa restaurant owner with strong community ties. The Sponsor provides a $5k balance the owner can use to cover dinners for farmers.

The motion can start with one trusted relationship:

```text
I got funds from John Deere to cover a few dinners for farmers.
If you have friends you would like to treat, bring them out.
I would love to host them and cover the cost thanks to John Deere.
```

| Layer | What happens |
|---|---|
| Sponsor | John Deere or dealer/co-op entity funds a local restaurant dinner balance. |
| Host | Restaurant owner with farmer trust invites through existing relationships. |
| Guest motion | A trusted farmer customer brings other farmers. |
| Spend | Owner uses the John Deere-funded card to cover dinner. |
| Community value | Farmers receive hospitality and a reason to gather. |
| Host value | Restaurant earns revenue, deepens relationships, and may win regulars. |
| Sponsor value | John Deere creates emotional goodwill inside a real farmer network. |

Proof should stay light: card spend, restaurant receipt, dinner happened, farmers attended, Sponsor was understood as the reason the meal was covered, Host note, optional photo, and whether new or lapsed customers came in.

## Use Case 2: Tapcart eCommerce Connector Dinner Network

Tapcart already sees ROI from events, but events are hard to run directly. They require travel, production, audience building, local marketing, and a constant need for novel new names.

Analog can fund eCommerce legends and dot connectors in multiple markets. Instead of Tapcart sending a team member everywhere, trusted local operators host dinners for the right founders, operators, agency leaders, and ecosystem people.

| Layer | What happens |
|---|---|
| Sponsor | Tapcart funds a dinner network for eCommerce operators and leaders. |
| Host | Local eCommerce legend, agency connector, operator, founder, or community builder. |
| Guest motion | Host personally curates the room and invites people Tapcart would struggle to reach cold. |
| Spend | Dinner costs are covered through event-scoped budget or spend card. |
| Community value | Operators get a useful room, not a weak sales event. |
| Host value | Dot connectors get funded and recognized for ecosystem labor. |
| Sponsor value | Tapcart creates authentic local impressions, learns markets, and builds relationship density. |

Markets can include Denver, Minneapolis, Chicago, London, and any city with real eCommerce community density.

## Product Setup

| Product object | John Deere setup | Tapcart setup |
|---|---|---|
| Sponsor | John Deere or dealer/co-op entity | Tapcart as a SaaS/Tech Company |
| Campaign | Farmer Dinner Fund | eCommerce Connector Dinner Network |
| Event sets | Iowa restaurant-hosted dinners | city lanes such as Denver, Minneapolis, Chicago, London |
| Hosts | restaurant owner with farmer trust | eCommerce legends, agencies, operators, founders, dot connectors |
| Host events | dinner with date, location, guest count, spend cap | curated dinner by market with guest thesis and spend cap |
| Spend card | $5k restaurant balance | dinner budget by Host event |
| Proof packet | receipt, attendance estimate, Host note, optional photo/tag, new-customer signal | guest quality, account fit, photos if appropriate, Host notes, follow-up meetings, market learning |
| Unlock | fund another dinner, expand to more restaurants/dealers, stop if trust/proof is weak | fund best Hosts, expand city lanes, create repeat salons, redesign guest thesis |

## Why Dinner Works

| Fit signal | Why it matters |
|---|---|
| Trusted Host | The room fills through relationships, not paid promotion. |
| Simple spend | One dinner can be governed with a card charge, receipt, and event note. |
| Sponsor generosity | The Sponsor makes a meaningful evening possible without turning it into a pitch. |
| Host upside | Restaurant owners and connectors earn real value, not exposure. |
| Light proof | Attendance quality, Host note, receipt, photos if natural, and follow-up can be enough for a first pilot. |

Tapcart should not judge only by attendance count. The better proof is attendee fit, qualified follow-up, partner signal, customer insight, ecosystem credibility, and lower Sponsor labor than a Sponsor-operated event.

## Use When / Avoid

Use dinner funds when the Sponsor wants trusted access, the Host has real community permission, the room can be curated through relationships, dinner spend is easy to govern, and the Sponsor is willing to be generous without turning the night into a pitch.

Avoid this motion when the Sponsor wants a captive sales pitch, the Host is only a vendor, the room needs heavy paid marketing, the Sponsor cannot tolerate light-touch attribution, or the proof standard requires more tracking than the social context can bear.
