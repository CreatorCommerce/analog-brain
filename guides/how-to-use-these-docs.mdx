These docs are designed for fast operator use. Start with the smallest page that frames the job, then pull canonical references only when you need product facts.

## If You Are Planning A Pilot

1. Read [Current Thesis](/strategy/current-thesis).
2. Pick the relevant sponsor perspective in [Sponsor Personas](/strategy/sponsor-personas).
3. Use [MVP Use Cases](/strategy/mvp-use-cases) to choose what the pilot should test.
4. Use [Audience Graph And Proof](/strategy/audience-graph-and-proof) if the pilot depends on RSVP, claims, imports, or attribution.
5. Score the market with [Testing Grid](/market/testing-grid).

## If You Are Activating Supply

1. Read [Host Models And Personas](/strategy/host-models-personas).
2. Read [Dot Connectors As Rockstars](/strategy/dot-connectors-as-rockstars) when the supply motion depends on local trust, taste, and community builders.
3. Read [Prestige Media And Funding Platform](/strategy/prestige-media-funding-platform) when the motion needs editorial status and on-platform fund access.
4. Decide whether you are accessing existing supply or creating new supply with [Supply Motion](/market/supply-motion).
5. Use [Event Prospecting Catalog](/market/event-prospecting-catalog) when supply exists but needs to be found, enriched, claimed, or prospected.
6. Use [City Leads And Host Unlocks](/market/city-leads-host-unlocks) when a local operator can create more options.
7. Use [Media And Owned IRL Engine](/strategy/media-owned-irl-engine) when city leads should turn field discovery into social, newsletter, print, video, and owned gatherings.
8. Use [Asset Need Intelligence](/market/asset-need-intelligence) when blockers repeat.

## If You Are Updating The Brain

Treat new user-provided use cases and corrections as binding. Use [Source Context Map](/references/source-context-map) to understand upstream ownership, then use [Update This Brain](/how-to/update-this-brain) to decide which pages must change.
