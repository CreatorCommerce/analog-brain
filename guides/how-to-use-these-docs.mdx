These docs are designed for fast operator use. Start with the smallest page that frames the job, then pull canonical references only when you need product facts.

## If You Are Planning A Pilot

1. Read [Current Thesis](/strategy/current-thesis).
2. Pick the relevant sponsor category in [Sponsor Personas](/strategy/sponsor-personas).
3. Use [MVP Use Cases](/strategy/mvp-use-cases) to choose what the pilot should test.
4. Score the market with [Testing Grid](/market/testing-grid).

## If You Are Activating Supply

1. Read [Host Models And Personas](/strategy/host-models-personas).
2. Decide whether you are accessing existing supply or creating new supply with [Supply Motion](/market/supply-motion).
3. Use [City Leads And Host Unlocks](/market/city-leads-host-unlocks) when a local operator can create more options.

## If You Are Using The Product

1. Read [Product Primitives](/references/product-primitives).
2. Read [Platform Logic](/references/platform-logic).
3. Follow the relevant how-to:

- [Setup Sponsor Campaign](/how-to/setup-sponsor-campaign)
- [Setup Host Event](/how-to/setup-host-event)
- [Run Creator-Led Activation](/how-to/run-creator-led-activation)
- [Location Activation](/how-to/location-activation)

## If You Are Updating The Brain

Treat new user-provided use cases and corrections as binding. Use [Update This Brain](/how-to/update-this-brain) to decide which pages must change.
