---
title: "Glossary"
description: "Canonical Analog vocabulary for sponsor, host, product, market-formation, and proof concepts."
pageType: "Reference"
---

| Term | Meaning |
|---|---|
| Sponsor | The product actor that funds a campaign. In the real world this can be an ecommerce brand, tech company, public fund, non-profit fund, civic sponsor, hospitality group, or other budget holder. |
| Host | The product actor that creates events and uses sponsor budgets. Hosts include creators, communities, organizers, space owners, local operators, and dot connectors when they originate or run an activation. |
| Brand | A common sponsor category and market buyer. Use `Sponsor` when describing the product object. |
| Campaign | Sponsor-owned funding container with budget, objective, proof requirements, and host relationships. |
| Collab | A sponsor-host relationship inside one campaign. It is the unit that grants a host access to sponsor budget. |
| Host event | Event created from the host side. The MVP currently expects a host event to use one sponsor budget. |
| Spend card | Temporary operational card that lets a host spend against sponsor-approved budget under limits, categories, and expiration. |
| Application portal | Sponsor-owned form surface where hosts submit structured sponsorship pitches. |
| Proof | Sponsor-visible evidence that the activation happened and deserves renewal. This can include spend, attendance, social proof, content, claims, or matched outcomes. |
| Market growth | Evidence that an activation increased category participation, content velocity, social conversation, recurring ritual strength, or sponsor-ready supply. |
| Spend integrity | Evidence that funded money was used responsibly, with budget discipline, receipts, vendor costs, and proof quality. |
| Unlock | A larger or next sponsorship allocation earned by market-growth proof and spend integrity. |
| Request for Events | Sponsor-side request for events that match a target audience, geography, category, proof requirement, and budget. |
| Request for Communities | Sponsor-side request for trusted communities that can originate the right IRL moment or recurring ritual. |
| Request for Sponsorship | Host-side package that asks sponsors to fund a specific event, community, venue, series, or activation. |
| Event Prospecting Catalog | Apollo-like internal market map of events, communities, venues, recurring rituals, and local activation surfaces. |
| City lead | Local operator who maps host supply, recruits or creates options, resolves missing assets, and helps more host-side unlocks become possible. |
| Dot connector | Host-side persona who may not own a venue or audience directly, but knows the local people, spaces, vendors, creators, and sponsors needed to assemble an activation. |
| Agent MCP | Model Context Protocol interface for external AI tools to read and operate Analog objects with approvals. |
| MCP attribution rail | Measurement, Commerce, and Proof layer that matches attendance or claim identifiers to sponsor-side outcomes safely. |
