---
title: "Glossary"
description: "Canonical Analog vocabulary for sponsor, host, product, market-formation, and proof concepts."
pageType: "Reference"
---

| Term | Meaning |
|---|---|
| Sponsor | The product actor that funds a campaign. Analog is built around three sponsor perspectives: eComm Brands that sell physical things to many, Tech Companies that sell digital things to fewer strategic users or accounts, and Public/Non-profit Funds that sell altruistic experiences and outcomes to the public. |
| Sponsor perspective | The strategic lens used to shape sponsor needs, proof, media framing, and city lead discovery. The three perspectives are eComm Brands, Tech Companies, and Public/Non-profit Funds. |
| Host | The product actor that creates events and uses sponsor budgets. Hosts include creators, communities, organizers, space owners, local operators, and dot connectors when they originate or run an activation. |
| Operating layer | Analog's role between sponsors and hosts: discover trusted supply, collaborate through programmable spend and operations, then measure proof for renewal. |
| Organization role | Current product bridge toward one organization model. A workspace can carry sponsor and host roles instead of being treated as a rigid account type forever. |
| Brand | A common sponsor category and market buyer. Use `Sponsor` when describing the product object. |
| Campaign | Sponsor-owned funding container with budget, objective, proof requirements, and host relationships. |
| Event set | Campaign subgroup for a market persona, activation lane, geography, special-request set, files/assets, and prospecting motion. |
| Collab | A sponsor-host relationship inside one campaign. It is the unit that grants a host access to sponsor budget. |
| Host event | Event created from the host side. The current wizard can create it with no sponsor or one selected sponsor budget, while the data model is future-shaped for richer sponsor summaries. |
| Hosted event page | Public event page attached to a host event. It can hold RSVP fields, ticket types, themes, custom CSS, and custom HTML. |
| Spend card | Temporary operational card that lets a host spend against sponsor-approved budget under limits, categories, and expiration. |
| Application portal | Sponsor-owned form surface where hosts submit structured sponsorship pitches. |
| Campaign outreach | Sponsor workspace motion for managed or future self-serve email/direct mail outbound tied to campaigns. |
| Event set prospecting | External prospecting run for one event set. It routes leads to the campaign application link. |
| Workspace file | Product file object for uploads, links, shared files, event set assets, application attachments, social content, and Analog Docs. |
| Analog Doc | Markdown document inside the files workspace. It can be manual, template-generated, or deep-research generated. |
| Deep research doc | Operator/AI-generated Analog Doc from a prompt and assignment context. |
| DM thread | Sponsor-host message thread available for active partnerships or pitch contexts. |
| Host task | Host event preparation or sponsor-required todo, including tasks sourced from event set special requests. |
| Asset request | Host request for a sponsor asset to be mailed. |
| Shipment log | Sponsor-side record of product samples, event supplies, printed materials, merch, signage, or other shipment references. |
| Proof | Sponsor-visible evidence that the activation happened and deserves renewal. This can include spend, attendance, social proof, content, claims, or matched outcomes. |
| Corporate underwriting | The proof loop that helps sponsors decide whether to deploy more capital based on spend stewardship, social growth, operator upside, category growth, brand growth, and community sentiment. |
| Audience graph | Privacy-safe intelligence layer built from hosted pages, attendance imports, check-ins, claims, and sponsor-side outcome matching. |
| Lookalike event | Event or recurring activation whose audience resembles a sponsor's best customers, target segment, or prior converting IRL audiences. |
| Existing customer overlap | Privacy-safe count or tier showing that attendees overlap with a sponsor's known customers, without exposing raw PII by default. |
| Market growth | Evidence that an activation increased category participation, content velocity, social conversation, recurring ritual strength, or sponsor-ready supply. |
| Media engine | The city-led content motion that turns field discovery into social posts, `The Connector`, YouTube documentaries, print volumes, and owned IRL gatherings. |
| The Connector | Analog's editorial brand for documenting the gathering economy through Substack, quarterly print editions, regional field notes, profiles, and event coverage. |
| Owned IRL | Analog-run gatherings such as happy hours, salons, dinners, walks, live recordings, and community-leader meetups that dogfood the product and create more field context. |
| Head of Content | The operator who owns editorial planning, channel strategy, sponsor perspective, production standards, city priorities, and coordination with the media team. |
| Media team | Contracted or assembled writing, design, copy, video editing, and publishing operators who produce content from city lead field submissions. |
| Spend integrity | Evidence that funded money was used responsibly, with budget discipline, receipts, vendor costs, and proof quality. |
| Spend stewardship | A broader underwriting signal that asks whether the partner used sponsor capital responsibly across budgets, vendors, receipts, anomalies, and unused funds. |
| Community sentiment | Evidence that the sponsor presence was additive, brand-safe, and welcomed by the community. |
| Operator upside | The host-side economic upside that lets dependable organizers earn for the work of gathering people. |
| Funded vocation | The target host state where recurring community work becomes funded, scoped, measurable, and renewable instead of unpaid side-hustle labor. |
| Unlock | A larger or next sponsorship allocation earned by market-growth proof and spend integrity. |
| Request for Events | Sponsor-side request for events that match a target audience, geography, category, proof requirement, and budget. |
| Request for Communities | Sponsor-side request for trusted communities that can originate the right IRL moment or recurring ritual. |
| Request for Sponsorship | Host-side package that asks sponsors to fund a specific event, community, venue, series, or activation. |
| Activation Opportunity | Connector- or operator-originated package that says "I can make this happen" before a sponsor campaign is fully formed. |
| Event Prospecting Catalog | Apollo-like internal market map of events, communities, venues, recurring rituals, and local activation surfaces. |
| Asset Need Intelligence | Map of missing communities, spaces, insurance, operators, vendors, proof, sponsors, and capital that block more IRL supply from existing. |
| City lead | Local operator who maps host supply, recruits or creates options, resolves missing assets, and helps more host-side unlocks become possible. |
| Dot connector | Host-side persona who may not own a venue or audience directly, but knows the local people, spaces, vendors, creators, and sponsors needed to assemble an activation. |
| Community builder | Person who repeatedly creates belonging, participation, and trust inside a community or local scene. |
| Connector profile | Sponsor-safe profile that makes a dot connector's network, taste, proof, city context, and activation ability legible. |
| Rockstar connector | Dot connector or community builder whose trust, local judgment, proof, media visibility, and economic upside make them a visible talent class inside the gathering economy. |
| CreatorCommerce Demand Bridge | Demand-side motion where CreatorCommerce or partner deployments qualify creator/affiliate-heavy brands for Analog IRL pilots. |
| Partner API | API-first strategy that lets community, club, venue, creator, ticketing, or operator platforms embed Analog sponsorship workflows without surrendering their own GUI. |
| Agent MCP | Model Context Protocol interface for external AI tools to read and operate Analog objects with approvals. |
| MCP attribution rail | Measurement, Commerce, and Proof layer that matches attendance or claim identifiers to sponsor-side outcomes safely. |
