## Before You Start

Write one sentence:

```text
This campaign tests whether [sponsor perspective: eComm Brand / Tech Company / Public-Non-profit Fund] can grow [category/audience/outcome] in [city/context] by funding [host type] to create [IRL activation].
```

## Campaign Fields

Capture:

- sponsor
- sponsor perspective: eComm Brand, Tech Company, or Public/Non-profit Fund
- campaign name
- budget
- optional end date
- objective
- mission
- market persona
- main value-add
- required todos or special requests
- eligible host type
- target geography
- proof requirement
- spend rules
- approved host and category boundaries
- renewal or unlock condition

## Set The Sponsor Perspective

Use the perspective to decide what the campaign must prove.

| Sponsor perspective | What the campaign is trying to prove |
|---|---|
| eComm Brand | Physical-product demand, trial, content, community sentiment, category growth, and sales-adjacent movement. |
| Tech Company | Digital-product adoption, strategic relationship density, qualified community access, enterprise or ecosystem relevance, and pipeline-adjacent learning. |
| Public/Non-profit Fund | Public-good participation, access, civic outcomes, local trust, and underfunded IRL supply creation. |

## Choose The Supply Path

| If the sponsor already knows the host | If the sponsor needs options |
|---|---|
| Create or accept that host directly | Use application portal, RFE, RFC, city lead, or prospecting catalog |

## Create The Event Set

Create an event set when the campaign needs a specific host lane, geography, category, or activation type.

Add:

- event set name
- market persona
- special requests
- files/assets hosts need
- prospecting plan only if external sourcing is needed

## Create The Intake Surface

If using inbound hosts, update the sponsor application portal so hosts submit comparable data:

- community or event name
- pitch
- expected attendees
- requested amount
- socials
- event date
- sponsor fit
- proof available

If the sponsor will mail product, supplies, or printed materials, require a package-capable mailing address.

## Source Hosts

Choose one:

- direct invite for known creators, communities, venues, or operators
- event set prospecting for off-platform communities
- city lead for local relationship-driven sourcing
- hosted/embedded form for sponsor-owned inbound
- partner API or partner list for embedded supply

## Approve Hosts

Approve a host only when:

- the sponsor fit is believable
- the host can execute
- the spend amount is specific
- the event or activation has clear timing
- proof can be captured
- the community is likely to welcome the sponsor
- the host has a clear operator-upside path if the program works

## After Approval

When approved, make sure the host can:

- see the sponsor budget
- create the host event
- access the spend card
- understand proof expectations
- request sponsor assets when needed
- see shared files/docs
- report blockers early
