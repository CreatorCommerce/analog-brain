Use this workflow when a sponsor has budget or a clear category goal but does not yet have enough host-side options.

Read this after [Capture Field Context](/how-to/capture-field-context) and [Event Prospecting Catalog](/market/event-prospecting-catalog). Those pages explain what to capture and how supply is structured.

## Inputs

Start with:

- sponsor
- campaign or campaign draft
- market persona
- commercial lane: eComm, SaaS/tech, civic/nonprofit, or political action
- city or geography
- category or audience
- budget range
- grant, raffle, or pitch-tool hook if the campaign needs supply capture
- acceptable sponsor presence
- proof requirement
- lane-specific controls such as public reporting, political compliance, privacy, or age/safety rules
- exclusions or trust constraints

## 1. Create Or Update The Event Set

In the campaign detail page, create an event set for the market/persona lane.

Capture:

- event set name
- market persona
- required host requests
- files or assets hosts need
- whether prospecting should be enabled

Keep special requests short. If there are more than five, the offer is probably too hard for hosts to parse.

## 2. Build The Prospect List

Use the [Event Prospecting Catalog](/market/event-prospecting-catalog), city lead research, partner lists, sponsor-known communities, public discovery, Market Agent context, and future public-safe API or Agent MCP search. Use [AI Memory And Context](/platform/ai-memory-and-context) for what an agent can safely retrieve and summarize.

For each prospect, record:

- name
- type: event, community, venue, creator, operator, or location
- owner/contact if known
- city/neighborhood
- category and audience
- reusable city supply insight across other Sponsor lanes
- cadence
- sponsor surfaces
- proof/data readiness
- visibility posture: public-safe opportunity, private/link-only, partner-scoped, compliance-sensitive, or opted out
- likely missing asset
- funding unlock: what gets better if money clears
- pitch readiness: whether they can show history, photos, reach, prior proof, or a clear ask
- trust or brand-safety notes
- next action

## 3. Split The List

| Bucket | Meaning | Next action |
|---|---|---|
| Sponsor-ready | Strong fit, known operator, proof can clear | Invite, application link, or direct collab |
| Needs enrichment | Looks promising but missing facts | Research, city lead call, partner lookup |
| Needs creation | Demand exists but host/space/operator is missing | City lead or grant motion |
| Not a fit | Weak sponsor permission, weak trust, or no proof path | Archive with reason |

## 4. Choose The Outreach Path

Choose the path that fits the source and visibility posture:

- Manual invite when the relationship is known.
- Application link when the sponsor wants standardized intake.
- Grant or raffle link when visible money should recruit qualified supply.
- Pitch tool when Hosts or connectors need help packaging themselves before a full application.
- Event set prospecting when Analog should run external outreach to off-platform communities.
- Agent MCP or API search when a custom workflow should query public-safe opportunity data and return a reviewed sourcing packet.

Event set prospecting should route all interested leads back to the campaign application link.
Agentic or API-assisted prospecting should not expose private attendee data, partner-private notes, sensitive financial/tax records, or compliance-sensitive public/political context.

## 5. Convert Responses

When a host responds:

1. Update or create the catalog record.
2. Ask for the missing proof or context.
3. Route them through the campaign application page or create a direct collab if already approved.
4. Attach relevant files/assets to the event set.
5. Score with the [MVP Use Cases](/strategy/mvp-use-cases) test scorecard.

## 6. Capture Market Learning

Every list should teach Analog what the market is missing.

Log:

- which records accepted, declined, ignored, or needed more information
- whether the blocker was host, space, insurance, operator, vendor, proof, sponsor fit, or capital
- whether city lead work could unlock more options
- which sponsor surfaces got traction
- whether the same supply could support eComm, SaaS/tech, civic/nonprofit, or political action with a different proof wrapper

Send repeated blockers back into the [Event Prospecting Catalog](/market/event-prospecting-catalog).
