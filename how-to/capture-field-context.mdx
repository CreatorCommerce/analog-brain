Use this workflow after a call, site visit, city lead sweep, Play Fair conversation, grant review, sponsor meeting, or manual activation.

Read this before [Build Event Prospecting List](/how-to/build-event-prospecting-list). Field capture turns messy context into objects; prospecting uses those objects to build supply.

## Capture The Smallest Useful Object

Do not leave context as loose notes. Convert it into one or more objects:

| Field signal | Product object |
|---|---|
| Existing community, event, venue, or recurring ritual | Catalog record |
| Sponsor has a clear demand signal | Campaign, event set, RFE, or RFC |
| Host wants money for a specific activation | Request for Sponsorship or campaign application |
| Host has an audience but no event yet | Host story, pitch doc, or RFC response |
| Space exists but needs programming | Location activation candidate |
| Vendor/operator can execute repeatably | Service provider or city supply record |
| Missing blocker appears repeatedly | Asset Need Intelligence note |
| Strong story, interview, or event footage | Media intake for social, `The Connector`, YouTube, print, or owned IRL |
| Community leaders need a convening point | Owned Analog gathering |

## Field Intake Questions

Ask enough to make the opportunity legible:

- Who owns the audience, space, or operator relationship?
- What gathers people now?
- How often does it happen?
- Who shows up?
- What sponsor presence would feel native?
- Which commercial lane fits best: eComm Brand, SaaS/Tech Company, Civic/Public-Non-profit Fund, or Political Action Sponsor?
- Is this same supply reusable across another lane with a different proof wrapper?
- What would sponsor money make better, cheaper, or possible?
- What proof can be captured?
- What media can be captured with permission: interview, audio, video, photo, field note, or recap?
- Which channel should this story feed: social, `The Connector`, YouTube, print, or owned IRL?
- What is missing: host, sponsor, space, insurance, vendor, operator, proof, or capital?
- What would make this repeat?
- What would make this culturally fragile?

## Sponsor-Safe Summary

Write a sponsor-safe summary in this shape:

```text
[Host/location/community] can reach [audience] through [format/cadence] in [city/context].
The best sponsor lane is [eComm Brand / SaaS/Tech Company / Civic/Public-Non-profit Fund / Political Action Sponsor].
The reusable city supply insight is [overlap with other lanes or local market asset].
Sponsor money would improve [consumer or operator value].
The strongest sponsor surfaces are [surfaces].
The proof path is [proof].
The biggest blocker is [missing asset].
The media opportunity is [story/interview/clip/newsletter/print/video/owned gathering].
Recommended next action: [catalog/enrich/apply/collab/grant/city lead/RFE/RFC].
```

When the note changes how Analog understands a city, category, or Host type, route it toward the AI memory loop in [AI Memory And Context](/platform/ai-memory-and-context): product record first, then reviewed recursive doc update, then future agent retrieval.

## Media Intake Summary

When the field note should go to the content team, include:

```text
City:
City lead:
Community or event:
People captured:
Media links:
Permission status:
Core story:
Sponsor perspective:
Commercial lane:
City supply insight:
Potential channel:
Product follow-up:
```

Submit the raw material in Slack so the Head of Content can triage and route it to the writing, design, or video team.

## Attach Files And Docs

When useful, create or update:

- an Analog Doc
- a deep research doc
- a pitch doc
- a campaign brief
- event set files/assets
- shared files for the sponsor or host
- media intake notes for the content team
- city or regional Connector planning docs

Use files to preserve context that will help the next operator move without starting over.

## Update The Docs When It Becomes Doctrine

Do not update docs for every local note. Update the operating docs when the field context changes a repeatable rule:

- a new use case works or fails
- a blocker appears repeatedly
- a sponsor persona needs a new rule
- a host model becomes more important
- a platform workflow should change
- a founder correction conflicts with existing docs

When the founder gives a use case or correction directly, treat it as binding and update conflicting docs in the same pass. Use [Documentation Contract (For AI Agents)](/guides/documentation-contract) for the canonical update workflow.
