## Inputs

Before creating the host event, confirm:

- host
- active sponsor/collab
- available allocation
- event start and end time
- approved spend categories
- event location or context
- proof requirements
- whether the event should start without sponsor backing
- whether an invite page, RSVP fields, ticket types, or claim flow is needed

## Steps

1. Open the host workspace.
2. Create a new event.
3. Select the sponsor budget, or choose no sponsor yet if the event is building credibility first.
4. Confirm the card and available amount.
5. Add timing and event details.
6. Configure the invite builder when attendee capture or a public page matters.
7. Review sponsor files, event set assets, special requests, and tasks.
8. Request mailed assets when downloads are not enough.
9. Use the spend card for approved operational spend.
10. Watch transactions during the event window.
11. Collect proof after the event.

## Proof Checklist

Collect:

- spend transactions
- receipts or vendor notes
- attendee count
- photo/video links
- sponsor-visible post or story mentions
- host notes on what worked and what blocked execution
- RSVP, check-in, import, or claim data when available
- task completion
- asset request or shipment status

## Renewal Notes

After the event, answer:

- Did the event create consumer or community value?
- Was sponsor money used responsibly?
- What proof would the sponsor trust?
- What would make the next activation cheaper or easier?
- Should the host unlock more budget?
