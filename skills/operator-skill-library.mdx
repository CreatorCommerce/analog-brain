## Promotion Priority

Promote these first:

1. `operational-agent-judgment`, because it prevents overbuilding.
2. `brand_intake`, because Sponsor demand must become structured briefs.
3. `campaign_pricing_and_roi`, because pricing and burn need disciplined language.
4. `event_matchmaking`, because market supply decisions recur constantly.
5. `activation_operations_and_integrations`, because manual excellence should become product primitives.

## Skill Improvement Rule

When a new operational use case appears, do not create a skill immediately.

First ask:

```text
Can one of these skills handle the job with a better input section, output format, or example?
```

If yes, improve the existing skill.

If no, use [Operational Agent Judgment](/agents/operational-agent-judgment) to decide whether the new work is a skill, agent, repo, service, or automation.
