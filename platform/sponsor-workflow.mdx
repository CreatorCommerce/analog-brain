Use this workflow when the sponsor already has a goal, budget, or campaign idea.

## 1. Create The Campaign

In the sponsor dashboard, create a campaign with:

- campaign name
- objective or sponsor thesis
- budget
- optional end date
- host visibility and eligibility rules if available

Write the objective as a testable market-growth question. Example: "Can LA recovery creators bring new runners into a brand-funded Saturday morning ritual?"

## 2. Publish Or Update The Application Portal

Use the application portal when the sponsor wants inbound host pitches.

Collect:

- community or host identity
- event or activation idea
- requested amount
- expected attendees
- relevant socials
- proof surfaces
- sponsor fit

## 3. Review Applications

Evaluate applications with the same questions used in [Testing Grid](/market/testing-grid):

- Does this host have trust?
- Is sponsorship native?
- Can spend be controlled?
- Can proof be captured?
- Could success unlock more budget?

## 4. Accept The Host

Accepting an application creates or links a host and creates a collab. The collab grants access to the sponsor budget.

## 5. Monitor The Activation

Use campaign detail and sponsor-host detail screens to review:

- sponsored hosts
- host-created events
- spend activity
- content or proof
- story and post mention signals when available

## 6. Decide The Unlock

After the activation, decide:

- renew
- increase allocation
- repeat with changes
- move budget to another host
- create more host supply through city leads or RFEs
