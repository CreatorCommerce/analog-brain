Use this workflow when the sponsor already has a goal, budget, or campaign idea.

## 1. Create The Campaign

In the sponsor dashboard, create a campaign with:

- campaign name
- objective or sponsor thesis
- budget
- optional end date
- mission
- market persona
- main value-add
- special requests
- host visibility and eligibility rules
- mailing-address requirement when product, supplies, or printed materials may ship

Write the objective as a testable market-growth question. Example: "Can LA recovery creators bring new runners into a brand-funded Saturday morning ritual?"

Also capture the sponsor's non-negotiables:

- brand safety
- approved host and category boundaries
- spend control and receipt requirements
- sentiment or social proof expectations
- category-growth evidence
- operational support needed from Analog

## 2. Create Event Sets

Use event sets to split the campaign into prospectable lanes.

Each event set should have:

- name
- market persona
- special requests
- files or assets hosts can use
- optional prospecting plan

Examples:

- Austin neighborhood tastings
- LA recovery run clubs
- NYC founder breakfasts
- Bowling venue access nights

## 3. Publish Or Update The Application Portal

Use the application portal when the sponsor wants inbound host pitches.

Collect:

- community or host identity
- event or activation idea
- requested amount
- expected attendees
- relevant socials
- proof surfaces
- sponsor fit

## 4. Prospect Or Invite Hosts

Choose the path:

- direct invite for known hosts
- application portal for standardized inbound
- event set prospecting for off-platform supply
- city lead when supply is relationship-driven
- catalog search when the sponsor needs more options

Use [Build Event Prospecting List](/how-to/build-event-prospecting-list) for external sourcing.

## 5. Review Applications

Evaluate applications with the same questions used in [Testing Grid](/market/testing-grid):

- Does this host have trust?
- Is sponsorship native?
- Can spend be controlled?
- Can proof be captured?
- Could success unlock more budget?

## 6. Accept The Host

Accepting an application creates or links a host and creates a collab. The collab grants access to the sponsor budget.

## 7. Monitor The Activation

Use campaign detail and sponsor-host detail screens to review:

- sponsored hosts
- host-created events
- spend activity
- content or proof
- story and post mention signals when available
- asset requests and shipments
- shared files and docs
- host tasks and special requests

## 8. Decide The Unlock

After the activation, decide:

- renew
- increase allocation
- repeat with changes
- move budget to another host
- create more host supply through city leads or RFEs

Use the underwriting lanes in [Proof And Spend](/platform/proof-and-spend): spend stewardship, social growth, operator upside, category growth, brand growth, and community sentiment.
