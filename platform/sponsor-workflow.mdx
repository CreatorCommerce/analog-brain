Use this workflow when the Sponsor already has a goal, budget, or campaign idea.

The MVP workflow should answer one commercial question: can Analog earn the right to deploy this Sponsor's community activation budget again?

Before setup, write the campaign thesis in one sentence:

```text
This campaign tests whether [Sponsor perspective] can grow [category/audience/outcome]
in [city/context] by funding [Host type] to create [IRL activation].
```

## 1. Create The Campaign

Create a campaign with:

- campaign name
- Sponsor perspective: eComm Brand, SaaS/Tech Company, Civic/Public-Non-profit Fund, or Political Action Sponsor
- objective or Sponsor thesis
- budget and optional end date
- mission, market persona, value-add, and special requests
- Host visibility and eligibility rules
- mailing-address requirement when product, supplies, or printed materials may ship

Write the objective as a testable market-growth question:

```text
Can this Sponsor fund this kind of Host,
in this market,
with this budget,
and receive proof strong enough to repeat?
```

Define the budget as a learning contract:

| Budget decision | What to clarify |
|---|---|
| Amount | How much capital can move into controlled field spend. |
| Allocation rule | One Host, one event set, dinner balance, city/category fund, access fund, or broader program. |
| Spend guardrails | Approved categories, timing, proof, assets, and operating boundaries. |
| Renewal rule | What evidence justifies another allocation, larger balance, or broader program. |

Choose the funding shape before writing the ask.

| Shape | Use when |
|---|---|
| Micro Grant | Small costs like coffee, libations, supplies, rentals, print, accessibility, or light space support would improve the gathering. |
| Macro Grant | A proven Host can run recurring activations, strategic rooms, or stronger community infrastructure. |
| Services/tools support | The blocker is insurance, docs, contracts, print, local designers, vendors, rentals, proof kits, or space support. |
| Embedded Host treasury | A trusted recurring Host or community business needs a larger operating balance and banking-quality tools instead of one-off spend cards. |
| Public fund | The budget needs eligibility, fraud/waste/abuse controls, public reporting, and defensible participation proof. |
| Political grassroots fund | The budget needs campaign/advocacy controls, compliant approved uses, field proof, privacy boundaries, and state/county/city/event rollups. |
| High-ticket underwriting | The Sponsor is considering six- or seven-figure IRL/community investment. |

For high-ticket programs, define the investment thesis, allocation model, tranches, proof gates, Host/operator underwriting, risk register, contracts/invoices, AI monitoring, executive reporting, and stop/renew criteria before launch. Do not force every dollar through a spend card.

Embedded Host treasury is a future product direction. The Sponsor benefit is not raw access to a private account. It is recipient-authorized investment-safety reporting: anomaly severity, spend stewardship, proof completeness, budget pacing, restricted-use risk, and renewal confidence.

Any grade derived from account activity is sensitive derived financial data. The recipient must authorize analysis, authorize summary sharing with the Sponsor, understand the funding-decision use, see the grade or key reasons, and have a way to dispute errors.

## 2. Create Event Sets

Use event sets to split the campaign into prospectable lanes.

Each event set should have:

- name
- market persona
- special requests
- files or assets Hosts can use
- optional prospecting plan

Examples: NYC founder breakfasts, LA recovery run clubs, Austin tastings, bowling venue access nights.

## 3. Publish The Application Portal

Use the application portal when the Sponsor wants inbound Host pitches.

Collect Host identity, activation idea, requested amount, expected attendees, relevant socials, proof surfaces, Sponsor fit, and support needs.

## 4. Share, Prospect, Or Invite Hosts

Choose the path:

| Path | Use when |
|---|---|
| Shared campaign/fund page | The goal is to let the market route qualified supply toward the opportunity. |
| Direct invite | Known Hosts already fit the brief. |
| Application portal | Inbound needs standard structure. |
| Event set prospecting | The right supply probably exists off-platform. |
| City lead | The supply is relationship-driven. |
| Catalog search | Sponsor needs more options from known supply. |

The shared ask should say who qualifies, what budget range exists, what spend is allowed, what proof is required, and where Hosts should apply.

Use [Build Event Prospecting List](/how-to/build-event-prospecting-list) for external sourcing.

## 5. Review Applications

Evaluate applications with the same questions used in the [MVP Use Cases](/strategy/mvp-use-cases) test scorecard:

- Does this Host have trust?
- Is sponsorship native?
- Can spend be controlled?
- Can proof be captured?
- Could success unlock more budget?

Frame proof by Sponsor perspective.

| Sponsor perspective | Campaign proof should emphasize |
|---|---|
| eComm Brand | product trial, content, category mentions, repeat participation, sales-adjacent signals |
| SaaS/Tech Company | strategic accounts, qualified attendees, community credibility, adoption moments, partner/customer conversations |
| Civic/Public-Non-profit Fund | access, participation, public benefit, community formation, local legitimacy, civic proof |
| Political Action Sponsor | compliant spend, local turnout or participation where permitted, volunteer/action proof, Host reliability, state/county/city/event allocation learning |

## 6. Accept, Monitor, Decide

Accepting an application creates or links a Host and creates a collab. The collab grants access to the Sponsor budget.

During the activation, review sponsored Hosts, Host events, spend activity, proof, asset requests, shipments, shared files/docs, tasks, and special requests.

After the activation, decide:

- renew
- increase allocation
- repeat with changes
- move budget to another Host
- create more Host supply through city leads, RFEs, or RFCs

Use [Proof And Spend](/platform/proof-and-spend) to judge spend stewardship, social growth, operator upside, category growth, brand growth, and community sentiment.

If the Sponsor would deploy more budget with the same rules, the convention is getting stronger. If belief requires custom service work, decide whether to productize the repeated step, price it as managed support, or narrow the use case.
