Proof and spend are the trust layer. They let sponsors increase allocations without treating every activation as a vague recap.

The seed-deck framing is stronger than reporting: Analog turns sponsorship proof and community growth into market-growth underwriting. The question is not only "what happened?" It is "should this sponsor deploy more capital through this host, city, category, or playbook?"

## Spend Infrastructure Focus

Spend infrastructure is the MVP's fintech enablement wedge. The point is not to expose financial complexity to hosts or sponsors. The point is to make budget deployment controllable enough that Analog can learn which spend conventions earn trust.

This is how spend infra becomes marketplace learning: every controlled dollar teaches which hosts are dependable, which sponsor asks pull supply, which proof is persuasive, and which product conventions should become defaults.

## Dogfood Spend Rule

Analog should use the same spend and proof standards on itself that it asks sponsors and Hosts to trust.

For the first six months, AnalogNYC spend should be categorized through Analog. Internal spend is not separate from product learning. It should teach the team which budget categories, merchant categories, receipts, proof requirements, and anomaly checks are clear enough to sell to customers.

| Internal KPI | Spend/proof requirement |
|---|---|
| Social growth | Attach spend to the owned IRL, media, or field moment that produced content and audience growth. |
| Category growth | Connect spend to the category, Host, ritual, or city supply it strengthened. |
| Community perspective | Capture Host notes, field notes, sentiment, referrals, and trust risks. |
| Direct response | Tie invitations to RSVPs, claims, replies, check-ins, attendance, or applications. |
| Spend transparency | Categorize NYC spend by campaign, event set, Host event, merchant, use case, and proof expectation. |

If Analog cannot explain its own NYC spend, it should not ask sponsors to believe the platform will explain theirs.

## Optimal Conventions To Discover

The MVP should treat spend infrastructure as instrumentation. The goal is to discover the smallest conventions that make sponsor budget deployment feel safe enough to repeat.

| Convention | Good default looks like | Bad default looks like |
|---|---|---|
| Budget unit | Campaign, event set, host allocation, dinner balance, city/category fund, or program lane is explicit. | Everyone describes the budget differently and proof cannot roll up. |
| Permission rule | Amount, category, event context, expiration, and proof expectation are clear before spend starts. | Host has money but unclear boundaries, creating sponsor anxiety. |
| Supply entry | Hosts apply, are invited, or are sourced against a visible fund/program ask. | Analog manually picks hosts without learning which market signal created quality supply. |
| Proof minimum | Receipt/spend, attendance or participation, host note, content, and sponsor-specific outcome are agreed up front. | Proof is invented after the activation and becomes a recap project. |
| Next-dollar logic | Sponsor knows what would trigger renewal, larger allocation, reallocation, or no-go. | Every renewal requires a new custom narrative. |

## Proof Lanes

| Lane | What it proves | Current state |
|---|---|---|
| Spend transactions | Money was used through approved operational workflows | MVP card and transaction model |
| Attendance or RSVP | People showed up or claimed interest | Hosted event page config exists; production capture/import is still early |
| Sponsor-visible UGC | The activation produced media proof | Post media and story mentions; no comments |
| Social growth | The host can grow category conversation | Strategy layer, not fully automated in MVP |
| Benefit claim | Sponsor created useful attendee value | Location and event claim flows are future-facing, but page/QR/print strategy is defined |
| Conversion match | Sponsor-side outcome happened later | Attribution workflow direction |
| Files and docs | The operator captured context, brief, proof, or research | Workspace files, Analog Docs, template docs, and deep research docs |
| Tasks and shipments | The host and sponsor completed required logistics | Host tasks, event set todos, asset requests, and shipment logs |

## Underwriting Lanes

| Lane | What it answers |
|---|---|
| Spend stewardship | Was capital used responsibly across budgets, receipts, vendors, approved uses, unused funds, and anomalies? |
| Social growth | Did content, posts, UGC, creator posts, community posts, and recap momentum reach the intended audience? |
| Community operator upside | Is the partner happy with the partnership, dependable, and earning enough for the work to continue? |
| Category growth | Did repeat participation, category mentions, new members, habit formation, or market activation improve? |
| Brand growth | Did awareness, benefit claims, signups, sales lift, attributed outcomes, or downstream conversion improve? |
| Community sentiment | Was the sponsor presence additive, brand-safe, and welcomed by comments, replies, trust signals, or qualitative feedback? |

## Spend Integrity

Spend integrity asks:

- Did the host spend within the approved limit?
- Were categories and merchants reasonable?
- Did timing line up with the event?
- Were receipts or vendor notes available?
- Were there anomalies?
- Did underspend create an opportunity to reward efficiency or reallocate budget?
- Did logistics and shipped/mailed assets arrive in time?
- Did sponsor-required tasks get completed before proof review?

## Sponsor-Visible UGC Rules

Show:

- image/video post mentions
- story mentions with media or source URLs
- manually uploaded proof where appropriate

Do not show:

- comments as sponsor UGC
- host-side analytics-heavy UGC grids
- raw private attendee data as a catalog or proof surface

## Unlock Logic

A host should earn a bigger or next allocation when:

- market-growth proof is stronger than expected
- spend stewardship is clean
- the community welcomed the sponsor presence
- the host earned enough operator upside to keep caring
- the sponsor can explain the value internally
- the activation can repeat
- product setup will get easier next time

Direct attribution can help, but it is not the only reason to unlock more budget. A credible underwriting signal can be enough when spend, sentiment, category growth, and repeatability are strong.

## Audience Graph Rule

When a pilot captures attendees, imports attendance, or runs a claim flow, write the proof summary in a privacy-safe way:

- audience count
- check-in or claim count
- data source
- consent source
- overlap or outcome match only where allowed
- confidence tier
- recommended renewal or next proof action
